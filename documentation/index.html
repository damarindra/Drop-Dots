Requirements and Instalation

Requirements:

  1. Android Studio with Android SDK, Available for Linux, Mac, Windows (http://developer.android.com/sdk/index.html)
  2. Java Development Kit (http://www.oracle.com/technetwork/java/javase/downloads/index.html)

For Eclipse user, I recommend migration to Android Studio. Eclipse is no longer supported by Google.

Android Studio

Import:

In this tutorial I'm using Ubuntu 14.04, but it should be similar with Mac or Windows User. Once you have extracted the zip file, Open Android Studio

When its loaded, you will see "Welcome to Android Studio. Click Import project (Eclipse ADT, Gradle, etc). New window (Select Eclipse or Gradle Project to import) will appear, then point to the core project Drop Dots and click OK.

Now Android Studio is going to build your project and then it will ask for Android SDK path. Just choose OK for automatically change the directory path of your Android SDK. Maybe it will take a long time, depend on your computer . Once Android Studio is finished their job, it looks like these.

Testing on Desktop

This game is build using LibGDX which is a powerfull Java Game Library. LibGDX is a Game Development Framework which can be export in multiplatform. This mean you can test on Desktop before you deploy it on Android . You can test your game on Desktop with full functionality, but feature like Admob, Leaderboard, Achievement, In App Purchase, etc not working on Desktop (not supported). If you want to test the feature, you must run it on real Android Device.

Testing on desktop is realy easy, just follow this step :

Open / Expand Project Desktop until you got DesktopLauncher. Right Click the DesktopLauncher and select Run.

Error? don't panic, Now Click on DesktopLauncher (see at top of window) then click Edit Configuration..

Browse your Working directory to DropDots/android/assets. Click OK and Apply. Now run again your DesktopLauncher, you can use the step number 2 or click Run right after DesktopLauncher (top of window). Then you will see your game.

Testing on Android

Testing on Android is very easy. Just change the Configuration to Android (Top Window), then right after Configuration you can find green button Run. Just click it and wait the project building. If you want to run on your real device, don't forget to connect your Android Device to your computer.

Once the project build has complete, new window will appear and you can choose your device. if you didn't connect with your device, you can use AVD. Now Android Studio building your debug apk to your device.

 

Project Structure

Let's take care with android, core, desktop, assets.

android : this project is your way to export to Android Platform. All services like Admob, Google Play Game Services, etc on here. I've doing magic with this project 

core : this project is your main game. This mean, all of game logic is on this project.

desktop : this project is your way to testing on Desktop. So? just leave it, you didn't need any change on here. 

assets : this folder placed inside of android project. All game assets is on here.

Configuration File

Most of you, is only want to change the color and ids, thats why I create class Constants where you can change all the value like colors, speed, ids, etc. Constants.java file inside core > java > yourpackage > utils > Constants.java.

Changing Game Name

Basically game name can be found in Constants.java. This will change your name game at Screen.

Then you need to change your package name. If you stay with default package name, you can't publish it to google play. Package Name is Unique. Okay now change with your own Package Name, let see, com.cgranule.dropdots. At middle is bassically company name, so change it with your own. The last one is your game name, change it. To change the package name you need to uncheck the option "Compact Empty Middle Package" by clicking gear icon at above of project directory.

Then right click on package name (cgranule and dropdots), choose Refactor > Rename. Maybe confirmation to refactoring is appear on bottom window, just choose Do Refactor.

Then go to AndroidManifest.xml and check if the package name has been change or no. If not, change it manually.

Finally go to android > res > values > strings.xml and change app_name. Thats name is appear bellow of the icon in Android Device.

I would also recommend to checking in build.gradle (android) if all package is same as your package name.

Changing Assets

All images and sounds used in game is placed on android > assets. If you want to change some images, you can go to desktop > assets_raw, on there you can found all images assets (be carefull, don't use new name. If you want to use new name, I recommend visit this link). "You said, all assets in android?" Yes, there all in android, image asset in desktop is raw, this assets is non-packed image. So you must pack it first. You can download TexturePacker from LibGDX on here. Once your download has done, open TexturePacker. You'll see this GUI.

Then click Open Project, point to DropDots folder and find packer. Open it!

Make sure the Input Directory and Output Directory are correct. Then click Pack'em all. Done, your assets image has been packed on android/assets.

ADS

Go to Constants.java and file ADMOB_BANNER_ANDROID and ADMOB_INTERSTITIAL_BANNER with your id.

Leaderboard and Achievement

Go to Constants.java and change the whole ID with your ID. You can create your own ID if you have Google PLay Developer Account.

In App Purchase

Open Constants.java, and fill GOOGLE_PLAY with your Key. You can find your key in Google Play Developer Console > All Application > Your Game > Services and APIs > YOUR LICENSE KEY FOR THIS APPLICATION. 

Then fill your In App Product key to Constants > IAP_NO_ADS. 

Google Analytics

Open Constants.java and fill GOOGLE_ANALYTICS_KEY with your key. You can create on here.

Disable Ads

Open AndroidLauncher.java and find setupInterstitialAds() and setupAdsBanner()  comment or delete that line code.

Disable In-app Purchase

Open Constants.java and find USING_IAP  then set the value to false.
